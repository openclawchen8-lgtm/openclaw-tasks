## Issues 彙整（按嚴重程度排序）

### 🔴 P0 — 完全無法登入（服務端崩潰）

#### #1717 — `/api/auth/signin/github` returns HTTP 500 — GitHub OAuth broken
- **URL**: https://github.com/openclaw/clawhub/issues/1717
- **日期**: 2026-04-17
- **狀態**: open
- **現象**: `/api/auth/signin/github` 回傳 HTTP 500，Vercel 和 Convex 後端都 500。`clawhub login` → 瀏覽器打開 → 500 導致重定向循環。
- **根因**: AUTH_GITHUB_ID / AUTH_GITHUB_SECRET 在 Convex 部署中缺失或過期。
- **影響**: 所有認證 CLI 操作被封鎖（publish, whoami, sync）。唯讀操作（search, explore）正常。
- **解法**:
  - PR #1720（jeffvsutherland）：`missingEnvVar()` 在模組初始化時檢查 env var，缺失時立即拋錯而非靜默回傳空字串，讓錯誤訊息明確指向 Convex dashboard。
  - 需在 Convex 部署中重新設定 AUTH_GITHUB_ID / AUTH_GITHUB_SECRET。

### 🟠 P1 — OAuth 回調配置錯誤

#### #1581 — GitHub OAuth callback redirects to 127.0.0.1 instead of Convex production URL
- **URL**: https://github.com/openclaw/clawhub/issues/1581
- **日期**: 2026-04-08
- **狀態**: open
- **現象**: GitHub 授權成功後，回調重定向到 `http://127.0.0.1:3000/api/auth/callback/github` 而非 `https://wry-manatee-359.convex.site/api/auth/callback/github`，導致 ERR_CONNECTION_REFUSED。
- **根因**: GitHub OAuth App 的 Authorization callback URL 註冊為開發 URL（127.0.0.1）而非 production URL。
- **解法**: 更新 GitHub OAuth App 配置中的 callback URL。

#### #1623 — Add issuer to GitHub authentication configuration（PR）
- **URL**: https://github.com/openclaw/clawhub/pull/1623
- **日期**: 2026-04-10
- **狀態**: open（PR 有語法錯誤未合併）
- **內容**: 在 `convex/auth.ts` 的 GitHub OAuth provider config 加入 `issuer: "https://github.com/login/oauth"` 以修復 NextAuth issuer 驗證失敗（#1619, #1621）。但 PR 有雙逗號 TypeScript 語法錯誤。

#### #1619 — Sign in failed. Please try again.（已關閉）
- **URL**: https://github.com/openclaw/clawhub/issues/1619
- **日期**: 2026-04-10
- **狀態**: closed（2026-04-15）
- **現象**: Web UI 和 CLI `clawhub login` 都出現 "Sign in failed. Please try again."。清除快取、換瀏覽器、撤銷 GitHub token 重連都無效。
- **相關**: 與 #1623 的 issuer 問題相關。

#### #1621 — Cannot signin. First time trying to publish（已關閉）
- **URL**: https://github.com/openclaw/clawhub/issues/1621
- **日期**: 2026-04-10
- **狀態**: closed

## 目前解法總結

1. **服務端 env var**：PR #1720 提供更好的錯誤提示，但需維護者在 Convex dashboard 重新設定 AUTH_GITHUB_ID / AUTH_GITHUB_SECRET。
2. **Callback URL**：需在 GitHub Developer Settings 更新 OAuth App 的 Authorization callback URL。
3. **Issuer**：PR #1623 修正方向正確但需修復語法錯誤後合併。
4. **軟刪除帳號**：目前無自動化解法，需管理員手動介入 hard-delete 舊記錄。長期應在帳號刪除流程中清除 OAuth mapping。
5. **Token redirect**：需統一 domain（clawhub.ai vs www.clawhub.ai），確保 cookie 在正確的 domain 下設定。
6. **基礎設施**：Convex WebSocket 需確保 production 部署不回傳 307 redirect。

---

## 結論

ClawHub GitHub OAuth 登入問題是**多層次**的：
- 最嚴重的 #1717（HTTP 500）是服務端配置問題，PR #1720 已提供 fail-fast 機制
- OAuth callback 配置（#1581）和 issuer 驗證（#1619/#1621）是部署配置問題
- 軟刪除帳號（#1116/#886/#1552）是資料層面的長期未修 bug
- Token redirect（#702）是 domain 不一致的邊際案例

多數問題的修復方向已明確，但合併進度和部署時程不明。
