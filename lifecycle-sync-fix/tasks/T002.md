# T002 - sync_issues_cron.py 反向同步機制

## 問題描述

**140 個 done/skip 任務缺少 GitHub Issue URL**，導致腳本無法關閉對應 issue。

腳本邏輯：
1. 掃描 done/skip 的 T*.md
2. 用正則提取 `github.com/.../issues/N` URL
3. 若找不到 URL → 跳過（無法關閉）

## 根因

早期完成的任務（建立 sync 機制前）沒有 issue URL；新任務若建立時未同步 GitHub 也會缺 URL。

## 修復方案

改用「反向同步」：
1. 用 `gh issue list --state open` 列出所有 OPEN issues
2. 用標題比對本地 done 任務
3. 標題匹配 → 關閉 issue

```python
# 偽代碼
open_issues = gh("issue list --state open --json number,title")
for issue in open_issues:
    if match_local_done_task(issue.title):
        gh(f"issue close {issue.number}")
```

## 驗收標準

- [✅] 支援 `--dry-run` 預覽
- [✅] 失敗時發 Telegram 通知
