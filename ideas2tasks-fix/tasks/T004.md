# T004 - 方案 A：Sub-agent 完成後強制更新 Task Status

## 基本資訊
- **Type**: enhancement
- **Assignee**: 寶寶
- **Priority**: high
- **Status**: pending

## 問題
Sub-agent 執行完任務後，改了程式碼但沒改 T*.md 的 Status，導致：
1. 任務狀態永遠停留在 pending
2. 下次掃描又被當成待處理 → 重複執行
3. idea 檔與 Tasks/ 目錄不同步

## 修法

### 1. Spawn Prompt 模板強制化
在 AGENTS.md 或 spawn 指令中加入強制規則：
```
任務完成後，你必須：
1. 更新 T*.md 的 Status 為 done（- **Status**: done）
2. 執行 python3 /Users/claw/.qclaw/workspace/skills/ideas2tasks/scripts/state_sync.py sync <project>
3. 回報完成摘要
```

### 2. 寶寶接收結果後主動同步
寶寶收到 sub-agent 結果時：
- 檢查對應 T*.md Status 是否已更新
- 若未更新，主動更新 + 執行 state_sync
- 同步 GitHub Issue 狀態（open → close）

### 3. task_callback.py 擴充
現有的 callback 只有 start/done/progress/fail，加一個 `status_sync` 動作：
```bash
python3 /Users/claw/scripts/task_callback.py <T*.md路徑> done --agent <名> --note "完成" --sync-state
```
`--sync-state` 觸發 state_sync.py 同步 idea 檔 + GitHub Issue
