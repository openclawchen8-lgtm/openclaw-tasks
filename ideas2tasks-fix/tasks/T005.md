## 問題
沒有機制定期檢查「程式碼已改完 → task 該標 done」，完全依賴 AI 自覺更新狀態。
方案 A 是即時防護，本方案是兜底稽核。

## 設計

### task_audit.py 核心邏輯
```python
def audit_tasks():
    for each pending task:
        1. 讀取 T*.md 的「完成條件」區段
        2. 比對實際狀態：
           - 檔案是否存在？（指定的產出路徑）
           - 關鍵代碼是否已寫入？（grep 特定 pattern）
           - GitHub Issue 是否已 closed？
        3. 判定結果：
           - ✅ auto_done：所有可驗證條件皆滿足 → 自動標 done
           - ⚠️ needs_review：部分條件滿足，需人工確認
           - ❌ still_pending：條件未滿足
        4. 對 auto_done 的 task：
           - 更新 T*.md Status: done
           - 執行 state_sync 同步 idea 檔
           - 關閉 GitHub Issue
        5. 輸出稽核報告
```

### 驗證規則（可擴充）
| 規則 | 檢查方式 | 可靠度 |
|------|---------|--------|
| 檔案存在 | Path.exists() | 高 |
| 關鍵代碼 | grep pattern in file | 中 |
| GitHub Issue closed | gh issue view --json state | 高 |
| 提到特定 commit | git log --grep | 高 |
| 無法自動判斷 | 標記 needs_review | — |

### 整合方式
- 作為 cron job 每日執行（如每日 22:00）
- 或手動觸發：`python3 task_audit.py [--project <name>] [--dry-run]`
- 輸出格式：JSON + 人類可讀摘要
- 稽核結果推送 Telegram（使用現有通知機制）
