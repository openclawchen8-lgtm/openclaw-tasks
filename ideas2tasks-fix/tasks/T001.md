# T001 - classify.py 只取 task.N 區段，忽略對話雜訊

## 問題
idea 檔可能夾帶整段對話歷史（如 gold-analysis-improve.txt），executor 把對話也拆成 task。

## 修法
classify.py 的 `parse_tasks_from_content()` 改為：
- 只取 `task.N` 開頭到下一個 `task.N`（或 EOF）之間的內容
- 不符合 `task.N` 格式的段落全部忽略
- 每個 task 的描述只取該 task 行之後、下一個 task 行之前的文字

## 測試
```bash
python3 -c "
from classify import parse_tasks_from_content
content = open('/Users/claw/Ideas/gold-analysis-improve.txt').read()
tasks = parse_tasks_from_content(content)
print(f'解析出 {len(tasks)} 個 task')
for t in tasks:
    print(f'  task.{t[\"num\"]}: {t[\"title\"][:60]}')
"
```
預期：只出現 2 個 task（task.1 和 task.2）
