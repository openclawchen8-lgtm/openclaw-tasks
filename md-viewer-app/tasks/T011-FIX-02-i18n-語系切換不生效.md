## 目標

實作 JS 層 i18n 機制，讓語言設定切換後所有 UI 文字同步更新。

## 根因

1. `window.applyLanguage` 未定義 → 語言切換無效
2. 所有 UI 文字硬編碼，無 i18n dictionary
3. `saveLanguage` Go bind 不觸發 `UpdateMenuLanguage()` → menubar 不會同步

## 實際修改

### `main.go` JS：i18n dictionary

```javascript
window.applyLanguage = function (lang) {
  var translations = {
    zhTW: { settings: "設定", theme: "佈景主題", zoom: "縮放靈敏度" /* ... */ },
    zhCN: {
      /* ... */
    },
    en: { settings: "Settings" /* ... */ },
    ja: {
      /* ... */
    },
    ko: {
      /* ... */
    },
  };
  var t = translations[lang] || translations["en"];
  // 更新 settings panel labels、drop zone、empty state 等
};
```

### `main.go` Go bind

```go
wv.Bind("saveLanguage", func(lang string) {
    if err := SetLanguage(lang); err != nil { /* error */ } else {
        wv.Eval(getConfigJS())
        UpdateMenuLanguage(lang)  // 同步更新 NSMenu
    }
})
```

### 啟動時

```go
UpdateMenuLanguage(currentConfig.Language)  // 在 SetupMenu 前呼叫
SetupMenu(...)
```

## 驗收標準（達成）

- [x] 設定面板切換語言後，settings panel / keyboard hint / empty state 立即更新
- [x] menubar 同步切換語言
- [x] 繁中/簡中/英/日/韓五語言皆有完整翻譯
