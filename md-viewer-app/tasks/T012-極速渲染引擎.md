## 目標

整合 swift-markdown 解析 AST，實現異步渲染，確保大型檔案（>1MB）不卡死 UI。

## 功能規格 (Technical Specs)

### 核心需求
1. **整合 swift-markdown**：使用 Swift 的 swift-markdown 套件解析 Markdown AST
2. **異步渲染**：大型檔案（>1MB）需分段加載，不阻塞 UI 執行緒
3. **GFM 支援**：支援標準 GitHub Flavored Markdown 語法
   - 表格
   - 刪除線
   - 自動鏈接
   - 任務清單

### 技術實作方向

**方案 A：純 Go + goldmark 優化**
- goldmark 已支援 GFM
- 使用 goroutine 異步渲染
- 大檔案分段：先渲染前 N 行，背景載入其餘
- 優點：不需引入 Swift，保持純 Go
- 缺點：可能不如 swift-markdown 快

**方案 B：Swift + swift-markdown + Go FFI**
- 使用 swift-markdown 解析（效能最佳）
- 透過 C FFI 或 gomobile 與 Go 整合
- 優點：真正的極速渲染
- 缺點：建置複雜度高，跨平台困難

**方案 C：WebView + JavaScript Markdown 解析器**
- 使用 marked.js 或類似套件
- 在 WebView 內直接解析
- 優點：實作簡單，WebView 已內建
- 缺點：可能不如原生快

### 推薦方案

**先試方案 A**（純 Go + goldmark 優化）：
1. 使用 goroutine 異步渲染
2. 大檔案（>1MB）分段載入
3. 先顯示前 500 行，背景載入其餘
4. 使用 progress indicator 顯示載入進度

## 使用說明 (User Guide)

打開 App 即可瞬間讀取內容。支援標準 GFM 語法，包含表格、刪除線與自動鏈接。
