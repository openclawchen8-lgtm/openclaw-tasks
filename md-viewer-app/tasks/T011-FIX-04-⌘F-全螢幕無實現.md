## 目標

讓 ⌘F 正確觸發全螢幕切換。

## 問題描述

⌘F 觸發 `menuFullscreen` selector，但無任何效果。

## 根因分析

1. **`self.targetWindow` 永遠是 nil**：`MDAppDelegate` 的 `targetWindow` property 宣告了但從未賦值。`menuFullscreen` 中 `[self.targetWindow toggleFullScreen:nil]` 發給 nil → ObjC no-op
2. **`MenuFullscreen` ID 與 `MenuAboutHelp` 衝突**：兩者都是 `= 10`（不影響功能，因為 fullscreen 由 ObjC 直接處理不走 Go callback，但需修正以避免混淆）

## 修改檔案

### `menu.m`

#### 修正 1：`setupMenuWithLang:` 中取得當前 window

在 `menuFullscreen` 改為直接取得 key window，而非依賴 property：

```objc
- (void)menuFullscreen {
    NSWindow *window = [NSApp keyWindow];
    if (window && [window respondsToSelector:@selector(toggleFullScreen:)]) {
        [window toggleFullScreen:nil];
    }
}
```

此方式比依賴 property 更可靠，因為 `keyWindow` 始終指向當前活動視窗。

#### 修正 2：移除 `targetWindow` property（不再需要）

```objc
// 刪除這行
@property (nonatomic, assign) NSWindow *targetWindow;
```

### `menu.go`

#### 修正 3：修正 ID 衝突

```go
// 修正前
MenuToggleSidebar = 9
MenuFullscreen    = 10
MenuAboutHelp     = 10  // 衝突！

// 修正後
MenuToggleSidebar = 9
MenuAboutHelp     = 10
MenuFullscreen    = 11
```
