## 目標

使用 FSEvents 監控當前開啟的 .md 檔案，偵測到 write 事件時自動觸發渲染更新。

## 功能規格 (Technical Specs)

### 核心需求
1. **檔案系統監控**：使用 macOS FSEvents 或 Go fsnotify
2. **即時更新**：偵測到 write 事件 → 重新載入並渲染
3. **防抖處理**：避免編輯器頻繁存檔造成多次渲染

### 技術實作方向

**方案 A：Go fsnotify（推薦）**
- 使用 `github.com/fsnotify/fsnotify`
- 跨平台支援（macOS/Windows/Linux）
- 優點：純 Go，實作簡單
- 缺點：可能不如 FSEvents 精準

**方案 B：macOS FSEvents**
- 使用 CGO 呼叫 FSEvents API
- 優點：原生 macOS，效能最佳
- 缺點：僅支援 macOS，建置複雜

**方案 C：Polling 輪詢**
- 定期檢查檔案修改時間
- 優點：最簡單
- 缺點：效能差，不即時

### 推薦方案

**使用方案 A**（fsnotify）：
```go
import "github.com/fsnotify/fsnotify"

watcher, _ := fsnotify.NewWatcher()
watcher.Add(filePath)

for {
    select {
    case event := <-watcher.Events:
        if event.Op&fsnotify.Write == fsnotify.Write {
            // 重新載入並渲染
            debounceReload()
        }
    }
}
```

### 防抖處理
- 使用 200ms debounce
- 編輯器存檔後 200ms 才觸發更新
- 避免頻繁渲染

## 使用說明 (User Guide)

搭配 Vim、Sublime 等編輯器使用。只要在編輯器按下 Cmd+S 存檔，預覽視窗會立即同步畫面。
