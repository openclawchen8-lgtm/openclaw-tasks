## 目標

在檔案重載（reload）時保持當前滾動位置，避免每次存檔後視圖跳回頂部。

## 現況

`loadFile` / `reloadFile` 時 `SetHtml` 重建 DOM，滾動位置必然歸零。

## 實作方式

### 讀取位置

在 `SetHtml` 之前讀取並儲存：

```javascript
var savedScrollY = window.scrollY;
```

### 寫回位置

HTML 注入完成後（在 `DOMContentLoaded` 或 body's `onload`）寫回：

```javascript
window.scrollTo({ top: savedScrollY, behavior: 'instant' });
```

### 時序問題

`SetHtml` 是非同步，必須確保 DOM 完全渲染後才能寫回 scrollY。建議：

```javascript
wv.SetHtml(html)
wv.Eval("window.addEventListener('load', function() { window.scrollTo(0, savedScrollY); }, { once: true });")
```

### 多個 reload 防護

避免連續 reload 時 scrollY 被舊值覆蓋，每次 reload 取當前值：

```go
wv.Eval("var savedY = window.scrollY;")
wv.SetHtml(newHTML)
wv.Eval("window.addEventListener('load', () => window.scrollTo(0, savedY), { once: true });")
```

## 驗收條件

- [ ] 外部編輯器存檔後，視圖保持在原本的滾動位置
- [ ] 快速連續存檔多次，仍保持最後位置
- [ ] 手動 ⌘R 重載也保持位置
- [ ] 切換檔案後，新檔從頂部開始
