## 目標

讓 md-viewer-webview 支援從 Finder 拖拉 .md 檔案到 app 視窗中開啟。

## 需求

### 功能描述
1. 用戶可以從 Finder 拖拉 .md 檔案到 md-viewer 視窗
2. 拖拉時顯示視覺提示（邊框高亮或 drop zone）
3. 放開後自動載入並渲染該 .md 檔案
4. 支援一次拖拉多個 .md 檔案（開啟第一個，或顯示選擇對話框）

### 技術實作方向

**方案 A：WebView HTML5 Drag & Drop**
- 在 webview 中監聽 `dragover` / `drop` 事件
- 透過 `webview.Bind` 暴露 Go 函數處理檔案路徑
- 優點：純 JS，跨平台
- 缺點：需要處理 security sandbox 限制

**方案 B：Go 原生 Drag & Drop**
- 使用 `golang.org/x/exp/shiny` 或第三方 drag-drop 套件
- 在 Go 層處理 drag event
- 優點：更可靠
- 缺點：可能需要額外依賴

**方案 C：macOS AppleScript**
- 使用 AppleScript 監聽 drag event
- 優點：原生體驗
- 缺點：複雜度高

### 推薦方案

**先試方案 A**（WebView HTML5 Drag & Drop）：
1. 在 HTML 中加入 drop zone overlay
2. JS 監聽 `dragover`（顯示提示）和 `drop`（取得檔案）
3. 透過 `webview.Bind("openFile", openFileFunc)` 暴露 Go 函數
4. JS 呼叫 `openFile(filePath)` 載入檔案

## 子任務

### T010-A — HTML Drop Zone UI
- 建立 drop zone overlay（半透明遮罩 + 提示文字）
- 拖拉時顯示，放開或離開時隱藏

### T010-B — JS Drag & Drop Event
- 監聽 `dragover` / `dragleave` / `drop` 事件
- 阻止預設行為（`e.preventDefault()`）
- 取得 `DataTransfer` 中的檔案路徑

### T010-C — Go 檔案處理函數
- 暴露 `openFile(filePath string)` 給 JS
- 讀取檔案 → goldmark 渲染 → 更新 webview

### T010-D — 驗收測試
- 從 Finder 拖拉單一 .md → 正確開啟
- 拖拉非 .md 檔案 → 顯示錯誤提示或忽略
- 拖拉多個 .md → 開啟第一個

## 產出

- `main.go` 更新（新增 `openFile` bind）
- `assets/template.html` 更新（drop zone UI）
- `assets/markdown.css` 更新（drop zone 樣式）
