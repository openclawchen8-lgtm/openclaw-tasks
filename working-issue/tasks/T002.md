# T002 - git 配置異動確認流程規範

## 概述
本文檔規範日後所有 git 配置相關異動的確認流程，確保任何涉及新增、修改、刪除的操作都經過確認後才執行。

---

## 一、適用範圍

| 操作類型 | 說明 |
|----------|------|
| git config | 全域或專案級配置 |
| .gitignore | 版本控制排除規則 |
| GitHub 設定 | Repository、Collaborator、Branch protection 等 |
| CI/CD 配置 | GitHub Actions、Workflow 檔案 |
| Hooks | Git hooks 脚本 |

---

## 二、確認流程

```
[意圖] → [擬定方案] → [提交確認] → [用户批准] → [執行] → [回報結果]
```

### 2.1 提交確認時必須包含

| 項目 | 說明 | 範例 |
|------|------|------|
| **執行方式** | 具體命令或操作步驟 | `git config --global user.email "new@email.com"` |
| **執行內容** | 變更的詳細說明 | 將主要郵箱從 old@email.com 改為 new@email.com |
| **影響範圍** | 哪些專案或全局會被影響 | 全域所有 local repos |
| **回滾方式** | 如何撤銷此次變更 | `git config --global user.email "old@email.com"` |

### 2.2 格式範本

```markdown
## Git 配置異動確認

### 執行方式
```bash
<具體命令>
```

### 執行內容
<詳細說明>

### 影響範圍
<影響的範圍>

### 回滾方式
```bash
<回滾命令>
```

---

⚠️ **注意**：此操作會影響全域配置，請確認後再執行。
```

---

## 三、「注意」提醒規則

### 3.1 必須標記「注意」的情境

| 情境 | 範例 |
|------|------|
| 全域配置變更 | ⚠️ **注意**：此為全域設定，會影響所有 future commits |
| 不可逆操作 | ⚠️ **注意**：刪除後無法復原，請確認已備份 |
| 遠端操作 | ⚠️ **注意**：此操作會推送到遠端倉庫 |
| 權限變更 | ⚠️ **注意**：會影響團隊成員的存取權限 |
| Hooks 修改 | ⚠️ **注意**：可能會影響團隊成員的本地操作 |

### 3.2 提醒格式

```markdown
⚠️ **注意**：<警告內容>

例如：
⚠️ **注意**：此修改會影響所有 future commits，建議先在測試環境驗證。
```

---

## 四、實際範例

### 範例 1：修改 Git user email

```
## Git 配置異動確認

### 執行方式
git config --global user.email "new@example.com"

### 執行內容
將 git 使用者郵箱從 "old@example.com" 改為 "new@example.com"

### 影響範圍
全域配置，未來所有 commits 都會使用新郵箱

### 回滾方式
git config --global user.email "old@example.com"

---

⚠️ **注意**：此為全域設定，會影響所有 future commits。請確認這是您預期的行為。
```

### 範例 2：新增 .gitignore 規則

```
## Git 配置異動確認

### 執行方式
在 .gitignore 新增以下行：
```
node_modules/
dist/
*.log
```

### 執行內容
新增 node_modules、dist、*.log 到忽略清單

### 影響範圍
僅影響當前專案，已追蹤的檔案不會被移除

### 回滾方式
從 .gitignore 移除對應行

---

⚠️ **注意**：新增忽略規則不會自動移除已追蹤的檔案。如需移除，請執行 `git rm --cached`。
```

---

## 五、驗證檢查清單

在提交確認前，確認以下項目：

- [ ] 執行方式已清楚列出
- [ ] 執行內容已詳細說明
- [ ] 影響範圍已標註
- [ ] 回滾方式已提供
- [ ] 「注意」提醒已正確放置（適用時）
- [ ] 確認這是必要操作（非必要則略過）

---

## 六、相關工具

| 工具 | 用途 |
|------|------|
| `git config --list` | 查看當前配置 |
| `git config --global --list` | 查看全域配置 |
| `git config --local --list` | 查看專案配置 |

---

## 七、結論

此規範確保所有 git 相關異動都經過審慎確認，避免意外變更導致問題。

** ключові моменти**：
1. 先確認再執行
2. 提供完整資訊（方式、內容、範圍、回滾）
3. 適時使用「注意」提醒
