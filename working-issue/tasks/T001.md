# T001 - Gateway 不穩定與任務中斷預防實作方式報告

## 概述
本文檔說明如何防止專案或任務啟動後，因 Gateway 不穩定或任何原因被中斷的預防實作方式。

---

## 一、現有防護機制盤點

### 1.1 Gateway 自動重啟
- **機制**：QClaw Electron 守護進程監控 Gateway 狀態
- **狀態**：✅ 已實作（PID 1230 正常運行）
- **驗證**：`ps aux | grep openclaw-gateway`

### 1.2 配置自動備份
- **位置**：`~/.qclaw/backup*/openclaw.*.json`
- **觸發**：每次 Gateway 重啟時自動快照
- **保留**：最近 10 份（~10天滾動）
- **功能**：配置異動追蹤 + 快速回滾

### 1.3 工作階段持久化
- **會話恢復**：支援中斷後恢復對話（sessions_spawn + resumeSessionId）
- **狀態儲存**：`~/.qclaw/sessions.json`

---

## 二、預防實作方式

### 2.1 短期防護（立即可用）

| 防護措施 | 說明 | 狀態 |
|----------|------|------|
| Cron 健康檢查 | 每 5 分鐘檢查 Gateway 狀態，異常自動重啟 | ✅ 已有 daily-task-review |
| 配置快照 | Gateway 重啟時自動備份 | ✅ 已有 |
| 會話定期存檔 | 長時間任務定期輸出 checkpoint | 🔄 待加強 |

### 2.2 中期防護（需開發）

| 防護措施 | 說明 | 優先級 |
|----------|------|--------|
| **任務中斷偵測** | Cron job 檢測任務狀態，中斷時重新喚醒 | 高 |
| **Checkpoints 機制** | 子任務完成時寫入 checkpoint，支援從中斷點恢復 | 高 |
| **Health API 監控** | 公開健康檢查端點，供外部監控系統使用 | 中 |

### 2.3 長期防護（架構改進）

| 防護措施 | 說明 | 優先級 |
|----------|------|--------|
| **分散式架構** | Gateway 多節點冗餘，避免單點故障 | 低 |
| **事件驅動恢复** | 異常事件觸發自動恢复流程 | 低 |

## 三、實作細節

### 3.1 健康檢查 Cron Job

```bash
# 每 5 分鐘檢查 Gateway
*/5 * * * * curl -sf http://localhost:19000/api/status > /dev/null || openclaw gateway restart
```

### 3.2 任務 Checkpoint 範例

```python
# tasks/checkpoint.py
def save_checkpoint(task_id, progress, artifacts):
    """定期存檔進度"""
    checkpoint = {
        "task_id": task_id,
        "progress": progress,
        "artifacts": artifacts,
        "timestamp": datetime.now().isoformat()
    }
    # 寫入 checkpoint 檔案
    with open(f"/Users/claw/Tasks/_checkpoints/{task_id}.json", "w") as f:
        json.dump(checkpoint, f)
```

### 3.3 任務恢復邏輯

```python
def resume_task(task_id):
    """從 checkpoint 恢復任務"""
    checkpoint_path = f"/Users/claw/Tasks/_checkpoints/{task_id}.json"
    if os.path.exists(checkpoint_path):
        with open(checkpoint_path) as f:
            return json.load(f)
    return None
```
