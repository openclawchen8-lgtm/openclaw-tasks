## 描述
開發預測模型優化決策準確率，使用歷史數據訓練模型。

## 模組說明

### 📐 `feature_engineering.py` — 特徵工程（原始價格 → 模型輸入）

**輸入**：每天的金價資料（收盤價 only）
**輸出**：42 個特徵欄位 + 1 個標籤欄位

| 類別 | 特徵舉例 |
|------|----------|
| 價格 | 5/20/60日均線、價格相對均線位置、1/5/20/60日報酬率 |
| 技術指標 | RSI(14)、MACD、布林帶位置、ATR |
| 動量 | 5/10/20日動量、ROC、CCI、KD |
| 波動率 | 5/10/20日歷史波動率、日均波幅 |
| 形態 | 價格相對20日高低、缺口標誌、成交量比 |
| 時間 | 星期幾、月份、季度 |

**標籤生成**：看未來 5 天的價格變化
- 漲 > 1% → `BUY (1)`
- 跌 > 1% → `SELL (-1)`
- 其他 → `HOLD (0)`

### 🧠 `model_trainer.py` — 模型訓練（特徵 → 訓練好的分類器）

**輸入**：特徵 DataFrame + 標籤
**輸出**：訓練好的 sklearn 模型（.pkl 檔）

**支援模型**：Random Forest（預設）、Gradient Boosting、Logistic Regression

**訓練流程**：
1. 時間序列分割（前 80% 訓練，後 20% 驗證，不用未來數據）
2. 標準化（StandardScaler）
3. 訓練模型（sklearn fit）
4. 交叉驗證（TimeSeriesSplit 5折）
5. 特徵重要性分析
6. 保存模型（`models/v1_random_forest.pkl`）+ 版本管理（`registry.json`）

**也有 `predict()` / `predict_proba()`** 可做即時預測。

### 📊 `model_evaluator.py` — 模型評估（模型到底有沒有用）

**輸入**：真實標籤 vs 預測標籤
**輸出**：評估報告

**分類指標**：Accuracy / Precision / Recall / F1 / ROC-AUC / 混淆矩陣
**錯誤分析**：哪些類別最常搞混、錯誤樣本的特徵差異
**自動告警**：準確率 < 50%、R² 為負、只預測一種類別

### 整體流程圖

```
台灣銀行歷史金價 (SQLite)
       ↓
feature_engineering.fit_transform()
  → 40+ 技術指標特徵 + 標籤（未來5天漲跌）
       ↓
model_trainer.train()
  → 時間序列分割 → RandomForest 訓練 → 交叉驗證
  → 保存 models/v1_random_forest.pkl
       ↓
model_evaluator.evaluate_classification()
  → Accuracy / F1 / 混淆矩陣 / 錯誤分析
       ↓
model_trainer.predict(今日特徵)
  → 輸出：BUY / SELL / HOLD + 機率
       ↓
❌ 這步沒接上 API（見下方現狀）
```

## Step 1 測試結果（2026-04-28）

### 環境修正
- `feature_engineering.py` **只用收盤價**（台灣銀行只有 local_sell，無 OHLCV）
- 修正：`ATR`、`CCI`、`KD` 改用收盤價估算版
- 修正：時間特徵 `dates.dayofweek` → `dates.dt.dayofweek`
- 修正：排除 `date` 欄位避免 sklearn dtype 衝突

### 訓練結果

| 指標 | 數值 | 意義 |
|------|------|------|
| 有效樣本 | 249 筆（309筆中） | 需 60 日滾動窗口 |
| 標籤分佈 | HOLD 44.2% / BUY 41.8% / SELL 14.1% | SELL 偏少 |
| 訓練準確率 | 97.99% | 虛高（過擬合）|
| 驗證準確率 | 98.00% | 假象（全驗證集皆 HOLD）|
| 交叉驗證（TSCV 5折）| **44.85% ± 22.79%** | ⚠️ **這個才是真的** |
| 即時預測（最新一天）| HOLD（75.61%）| 保險牌 |

### 核心問題

- **CV 只有 44.85%**，跟隨機猜（33%）差不多，模型根本沒學到規律
- 驗證集只有 HOLD（49/50），模型只要全猜 HOLD 就能 98%，但這沒有任何預測能力
- **SELL 完全放棄**（0%）：F1=0，模型幾乎不預測 SELL
- 根因：42 個特徵 + 249 個樣本太稀疏；5 天後漲跌的信號本身就很難預測

### Top 5 重要特徵
1. `bb_lower`（布林帶下軌）
2. `sma_long`（60日均線）
3. `macd_signal`（MACD 信號線）
4. `return_60d`（60日報酬率）
5. `price_vs_sma_long`（價格相對均線）
