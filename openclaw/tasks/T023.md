How to create pre-defined members with agents's defined md files without startup openclaw.

### 核心結論

**不需要啟動 OpenClaw 也能建立完整 sub-agent**。原理是 OpenClaw 的 sub-agent 透過三層配置驅動：

1. **身份層** — workspace 的 `.md` 檔案（IDENTITY.md / SOUL.md / USER.md 等）
2. **註冊層** — `openclaw.json` 的 `agents.list`
3. **模型層** — `agents/<id>/agent/models.json` + `auth-profiles.json`

這些都是純文字檔案，任何時間都可以直接寫入，不需要 OpenClaw 在線。

### 快速流程

```bash
# 1. 建立 workspace + 所有 md 檔案
mkdir -p ~/.qclaw/workspace-newbot
# 寫入 IDENTITY.md, SOUL.md, USER.md, AGENTS.md 等

# 2. 建立 agentDir 並複製配置
mkdir -p ~/.qclaw/agents/agent-newbot/agent
cp ~/.qclaw/agents/main/agent/auth-profiles.json ~/.qclaw/agents/agent-newbot/agent/
cp ~/.qclaw/agents/main/agent/models.json ~/.qclaw/agents/agent-newbot/agent/

# 3. 用 gateway config.patch 註冊到 openclaw.json
# 4. 重啟 Gateway
```

### 免重啟驗證法

直接在新 agent 的 workspace 寫好 md 檔案 → 在 QClaw UI 新開一個與該 agent 的 DM → OpenClaw 即時讀取 workspace 驗證配置 → 確認無誤後再重啟 Gateway。
