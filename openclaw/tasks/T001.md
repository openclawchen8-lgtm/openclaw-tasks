## 目標
讓 QClaw（Port 28789）與 OpenClaw（Port 11434）同時運行於同一台 Mac，OpenClaw 使用本地 Ollama LLM。

---

## 當前狀態

| 組件 | Port | 狀態 |
|------|------|------|
| QClaw Gateway | 28789 | ✅ 運行中 (PID 3834) |
| OpenClaw（新實例） | 11434 | ⏳ 待啟動 |

---

## 完整執行指令稿

### 1. 確保 Ollama 運行（Local LLM）

```bash
# 檢查 Ollama 是否運行
pgrep -fl ollama

# 若未運行，啟動 Ollama
ollama serve &
# 或
brew services start ollama

# 驗證 Ollama 可用
curl http://localhost:11434/api/tags

### 2. 建立 OpenClaw 專用配置目錄

```
# 建立獨立的 OpenClaw 配置目錄
mkdir -p ~/openclaw-local
cd ~/openclaw-local
```

### 3. 配置 Local LLM（Ollama）
```
# 編輯 openclaw.json，添加 Ollama 配置
cd ~/openclaw-local
cat > openclaw.json << 'EOF'
{
  "gateway": {
    "mode": "local",
    "port": 11434,
    "bind": "loopback",
    "auth": {
      "mode": "token",
      "token": "local-ollama-token-2026"
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "ollama/llama3.2",
        "fallbacks": [
          "ollama/llama3.2:1b",
          "ollama/codellama"
        ]
      }
    }
  },
  "auth": {
    "profiles": {
      "ollama": {
        "provider": "ollama",
        "mode": "url",
        "url": "http://localhost:11434"
      }
    }
  }
}
EOF
```

### 4. 啟動 OpenClaw（新實例）
```
cd ~/openclaw-local

# 使用自定義配置啟動 Gateway
openclaw gateway start --config ~/openclaw-local/openclaw.json
```

### 5. 驗證雙開成功
```
# 檢查兩個 Gateway 都正常運行
lsof -i :28789 | grep LISTEN  # QClaw
lsof -i :11434 | grep LISTEN  # OpenClaw

# 測試 OpenClaw API
curl -H "Authorization: Bearer local-ollama-token-2026" http://localhost:11434/health

# 測試 Local LLM
curl -X POST http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "Hello",
  "stream": false
}'
```
