## 目標
收集並量化三個案例的真實運行數據，以成果證明系統的實際價值。

## 數據需求

### T006-C1：黃金監控系統
從 `gold_monitor_pro.db` 查詢：
- 運行天數（第一筆資料時間 → 現在）
- 告警觸發次數（is_alert=1 的記錄數）
- 覆蓋交易日數
- 資料庫總記錄數（gold / silver / platinum 各多少）

指令：
```bash
sqlite3 ~/.qclaw/gold_monitor_pro.db "SELECT COUNT(*), MIN(timestamp), MAX(timestamp) FROM price_history;"
```

### T006-C2：備份自動化
從 git log / 備份記錄查詢：
- 備份執行次數
- 每次備份的 tar.gz 大小
- GitHub Releases 上傳記錄數
- 備份覆蓋的目錄

### T006-C3：ideas2tasks 工作流
從專案目錄查詢：
- 已處理的 idea 數量
- 建立的有效 tasks 數量
- 涵蓋的專案數量
- 自系統啟用以來的運行次數

## 輸出格式
JSON 檔案 `outputs/case-stats.json`，同時寫入對應的案例文件中。

## 備註
需先確認資料來源位置（db路徑、git log、檔案時間戳等）
