# 案例三：ideas2tasks 自動化工作流

## 專案概述

好點子總是一閃而過，傳統做法是丢進備忘錄然後遺忘。ideas2tasks 讓點子從「想到」到「執行」零阻力——放在 Ideas 目錄的想法，AI 每小時自動掃描、分類、建立 GitHub Issues 與任務，追蹤到底。

---

## 技術架構

| 層次 | 技術 |
|------|------|
| 觸發 | cron（每小時）|
| 掃描引擎 | Python（正則 + 模糊匹配）|
| 格式解析 | 支援 `task.N`、`[ ]`、`## Todo` 等5種標記 |
| 任務建立 | GitHub API → Issues + Projects |
| 狀態同步 | 三層防護（Ideas → Tasks → GitHub）|
| 通知 | Telegram（有新任務即時通知）|

---

## 核心功能

- **多格式解析**：支援 `task.N`、`[ ]`（Checkbox）、`## Todo`、中文「任務」等多種格式
- **自動分類**：根據關鍵詞（自動化、金鑰、測試等）自動指派專案
- **三層狀態同步**：Ideas → Tasks → GitHub Issues，三個地方狀態始終一致
- **去重機制**：精確比對 + 正規化比對 + 相似度 >0.8 跳過，避免重複建任務
- **自動歸檔**：所有 tasks 完成的 idea 自動移入 `_done/` 目錄
- **GitHub Projects 同步**：任務自動出現在看板上，視覺化管理進度

---

## 亮點設計

1. **去重三層防護**：精確標題一致跳過 → 移除 T001/URL/特殊符號正規化後一致跳過 → 相似度 >0.8 跳過。同一個想法跑幾十次都不會重複建立任務。

2. **模糊格式支援**：不強迫用戶記格式，無論是 `task.1 done`、`[x]`、`## 待辦`、`任務` 都認得出來。

3. **ground truth 原則**：Tasks/ 目錄狀態為唯一事實來源，任何地方狀態不一致都以它為準。

## 成果數據

| 指標 | 數值 |
|------|------|
| 已處理想法 | 21 個（歸檔）|
| 有效任務建立 | 184 個（跨多個專案）|
| 涵蓋專案數 | 28+ 個 |
| 同步頻率 | 每小時自動掃描 |
| 現有運行中想法 | 2 個（黃金監控優化、GitHub 資料審查）|

## 應用場景

- 经常有好點子但執行力不足的人
- 需要系統化管理任務的 Project Manager
- 想讓 AI 幫你把「想到」轉成「做到」的行動派
