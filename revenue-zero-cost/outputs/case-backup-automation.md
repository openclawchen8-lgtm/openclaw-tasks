# 案例二：OpenClaw 備份自動化

## 專案概述

OpenClaw 是日常工作的核心工具，一旦資料丢失代價極高。備份自動化系統每 2 小時自動將所有重要資料打包上傳到 GitHub Releases，並在本地保留完整還原能力，全程零人為干預。

---

## 技術架構

| 層次 | 技術 |
|------|------|
| 打包工具 | tar.gz（純 Bash）|
| 版本控制 | Git + GitHub Releases API |
| 排程 | cron（每 2 小時）|
| 通知 | Telegram Bot（成功/失敗即時通知）|
| 清理策略 | 保留最近 N 份，自動刪除過期存檔 |
| 排除規則 | `.git`、`node_modules`、臨時檔案 |

---

## 核心功能

- **全自動打包**：Skills、Howto、Tasks、Agents、Workspace 全涵蓋
- **增量備份**：每次僅打包變動的檔案
- **GitHub Releases 上傳**：永久存檔，支援任意時間點下載
- **失敗告警**：上傳失敗即時 Telegram 通知
- **清理策略**：自動刪除 30 天前的舊存檔，避免佔用空間
- **備份大小追蹤**：每次記錄 tar.gz 大小，趨勢一目了然

---

## 亮點設計

1. **排除敏感目錄**：`.openclaw`、Session logs 等明確排除，避免備份滲入隱私資料。

2. **失敗即通知**：成功靜默、失敗發告警，符合「正常就要安靜」的运维原則。

3. **容量監控**：每次備份記錄大小，tar.gz 約 130MB，異常增大可提前發現。

---

## 成果數據

| 指標 | 數值 |
|------|------|
| 備份頻率 | 每 2 小時一次 |
| 典型備份大小 | 約 130 MB（tar.gz 壓縮後）|
| 覆蓋目錄 | Skills、Howto、Tasks、Agents、Workspace |
| 運行天數 | 2026-04-12 起（持續）|

---

## 應用場景

- OpenClaw 重度用戶，不想手動備份
- 有多台設備需要同步設定的人
- 想確保創意記錄（Ideas）和任務系統永不丢失的 Planner
