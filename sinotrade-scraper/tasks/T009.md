# T009 - Python 環境規範重構

## 背景
依據 qclaw-env SKILL.md 的 Python 環境規範，sinotrade_scraper.py 存在多處不符合規範的問題。

## 發現的問題

### P0 - 必須修復
1. **缺少 pyproject.toml** — 無專案元數據聲明
2. **硬編碼路徑** — 
   - `CHROME = "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"`
   - `HISTORY_FILE = os.path.expanduser("~/.qclaw/sinotrade_history.json")`
   - `TELEGRAM_CONFIG = os.path.expanduser("~/.qclaw/gold_monitor_config.json")`

### P1 - 建議修復
3. **缺少包結構** — 單文件腳本，無 `__init__.py`，不利於擴展和測試

### P2 - 可選
4. **備份檔殘留** — `/Users/claw/scripts/sinotrade_scraper.py.bak` 應刪除
5. **無 uv 整合** — 缺少 `.python-version` 和 `uv.lock`

## 新舊版本運行方式

### 舊版（單文件腳本）
```bash
# 直接執行
python3 /Users/claw/scripts/sinotrade_scraper.py --telegram
```

### 新版（Python package）
```bash
# 方式一：安裝後直接運行（推薦）
pip3 install -e /Users/claw/.qclaw/workspace/skills/sinotrade-scraper
python3 -m sinotrade_scraper --telegram

# 方式二：設置 PYTHONPATH（臨時測試）
PYTHONPATH=/Users/claw/.qclaw/workspace/skills/sinotrade-scraper \
  python3 -m sinotrade_scraper --telegram
```

### 兩者差異
| 項目 | 舊版 | 新版 |
|------|------|------|
| 位置 | `/Users/claw/scripts/` | `/Users/claw/.qclaw/workspace/skills/sinotrade-scraper/` |
| 結構 | 單文件 `.py` | package 目錄 |
| 配置 | 硬編碼路徑 | 環境變數 / 配置檔 |
| 運行 | 直接執行 | `pip install -e .` 後運行 |
| 擴展 | 難以擴展 | 模組化，易於測試 |
