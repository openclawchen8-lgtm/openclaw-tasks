## 架構設計

### 新增模式：`--sync-github`
```
executor.py --sync-github

掃描 /Users/claw/Tasks/*/tasks/T*.md
  ↓
讀取每個 T*.md 的 frontmatter
  ├─ 有 github_issue: → skip
  └─ 無 github_issue: → gh issue create → 補上 URL → 更新 frontmatter
```

### 與現有模式並行
- `--github`：走 lifecycle_status.json（舊流程，保留）
- `--sync-github`：掃 T*.md 直接同步（新流程，取代 `--github` cron）
