## 背景

目前 `gold_bot_history.py` 有三個問題：

1. **DB 不存在時崩掉**：`FileNotFoundError`（沒有自動建立 DB + schema）
2. **gap-filling 缺失**：如果某些日期在 DB 裡是空的，腳本不會補
3. **同步邏輯不明確**：用戶不清楚「同步」是什麼意思

## 需求

### 1. 自動建立 DB + schema（第一次執行時）
- 如果 `~/.qclaw/gold_monitor_pro.db` 不存在 → 自動建立
- 如果 schema 缺 table/index → 自動建立
- **不需要用戶手動干預**

### 2. Gap-filling（增量同步）
- 每次執行時，檢查 DB 中已有哪些日期
- 只補「DB 沒有的日期」，不覆蓋已有資料
- `--force` flag：可選強制覆蓋已存在的日期

### 3. 明確同步邏輯文件化
- 腳本頂部註釋說明：
  - 「同步」= 從台灣銀行官網抓滾動一年數據，寫入 DB（已存在日期跳過）
  - 網站 URL：`https://rate.bot.com.tw/gold/chart/year/TWD`
  - 網站資料範圍：約 1 年（目前 2025-04-21 ~ 2026-04-17，約 244 筆）

### 4. 命令列介面
```bash
python3 gold_bot_history.py              # 同步（抓網站+寫入DB）
python3 gold_bot_history.py --dry-run     # 預覽要寫入的筆數
python3 gold_bot_history.py --stats       # 只看資料庫現況
python3 gold_bot_history.py --force       # 強制覆蓋已存在的日期
python3 gold_bot_history.py --init        # 強制重建（清除 gold data 後重新匯入）
```
