## 背景

豪提出 gold_monitor_pro 系統的架構調整方向：

### 台灣銀行 URL 模式（新發現）

| URL | 內容 |
|-----|------|
| `https://rate.bot.com.tw/gold/chart/YYYY-MM/TWD` | 指定年月的每日資料 |
| `https://rate.bot.com.tw/gold/chart/day/TWD` | 最近一個營業日 |

→ 可用於滾動抓所有可用歷史（不限於網站首頁顯示的一年）

### gold_monitor_pro.py 架構調整

**現況問題**：
- gold_monitor_pro.py 同時做兩件事：寫入 SQLite（daily close） + 即時檢查（tmp file）→ 職責混淆

**目標架構**：

```
gold_monitor_pro.py（職責：即時檢查，讀 SQLite 取基準）
       ↓
gold_monitor_pro 啟動
       ↓
[1] 自動跑 gold_bot_history.py → SQLite backfill 到前一個營業日
[2] 從 SQLite 取最近營業日收盤價（基準）
[3] 即時檢查用 tmp file（每次抓新資料後馬上比對，不留 state）
[4] tmp file 每次啟動務必清空 → 確保數據乾淨
       ↓
每日 22:00 daily close
       ↓
gold_bot_history.py --sync → SQLite 更新（不重寫歷史）
```

**職責分離**：
- `gold_bot_history.py`：唯一寫入 SQLite 的地方（backfill + daily close 更新）
- `gold_monitor_pro.py`：只讀 SQLite 取基準 + 用 tmp file 做即時比較，不寫 SQLite

### tmp file 設計

- 位置：`/tmp/gold_monitor_check.json` 或 similar
- 每次 gold_monitor_pro 啟動時清空
- 即時檢查：抓新資料 → 寫 tmp → 比對 tmp 內的上一筆 → 發通知 → 更新 tmp
- 發送通知後即時刪除 tmp file，不累積狀態

## 預計產出

- `gold_monitor_pro.py`（重構版）：移除 SQLite 寫入、啟動時清空 tmp file、參考 SQLite 基準
- 更新 `gold_bot_history.py`：支援 `--YYYY-MM` 指定月份抓取、backfill 模式
