## 任務目標

技術分析 API（`/api/technicals`）目前讀 SQLite，但 SQLite 為空，導致指標計算失敗。

改從 Yahoo Finance 取得黃金現貨（GC=F 或 GLD）的歷史日 K 數據，補足 60+ 根 K 線後再交給 `TechnicalAnalysisAgent` 計算。

## 需求規格

### 資料來源
- **首選**：Yahoo Finance `GC=F`（黃金期貨現貨）
  - Symbol: `GC=F`（CME 黃金期貨現貨）
  - 或備選：`GLD`（SPDR 黃金 ETF， Yahoo Finance 有）
- 介面：`yfinance` Python 套件，free API，不需要 key
- 取得：最近 90 個交易日的收盤價（`Close` 欄位）
- 目標：60+ 根有效日 K

### API Endpoint 修改（`/api/technicals`）
1. 先嘗試從 Yahoo Finance 取 90 天歷史
2. 若取到 ≥60 筆 → 正常走 `TechnicalAnalysisAgent.analyze()`
3. 若取到 <60 筆（如網路問題）→ 回 `{"error": "歷史資料不足（Yahoo Finance 取到 N 筆，需 60+）"}`
4. 資料來源：Yahoo Finance > SQLite > JSON（降級順序）

### 不修改的事
- `TechnicalAnalysisAgent` 的指標計算邏輯（不動週期、不動門檻）
- 前端 `TechnicalsPage.tsx`（假設已經是正確格式）

## 備註

- `TechnicalAnalysisAgent` 需要 60 筆是因為 MA60、RSI(14)、MACD(26)，這些是數學定義不能隨便改
- Yahoo Finance 的 `Close` = 每日收盤價（適合日線技術分析）
- 如果 GC=F 取不到，備用方案是 GLD（黃金現貨 ETF，日成交量夠大）
